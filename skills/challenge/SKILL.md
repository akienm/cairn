# /challenge — harden the thinking while it still forms

You are firing the **challenge nexus**: an adversarial question set aimed at
UNSETTLED thinking, *before* a claim is made. The pose is adversarial but not
destructive — the goal is to harden the design, or to catch that it should change,
while changing it is still cheap. Same pose as the advisor and as `/sorted`'s inward
check, different placement: this one runs *before the claim*.

The charter lives beside this file in `intention+why.json`.

## Fire these questions

1. **Better approach?** Is there a cleaner, simpler, or smaller way to reach the
   same intent? Name it, or say honestly that you looked and this is it.
2. **Prior art?** Has this been solved already — earlier in Cairn, in the quarry
   (read-only), or in the field/state-of-the-art? Re-deriving the settled is a
   defect (Law 1). If it exists, cite it.
3. **Hidden assumption?** What is this taking for granted that could be false? State
   the load-bearing assumptions out loud so they can be checked.
4. **Real collision?** Does this conflict with an existing intention, Law, or
   decision — and does it *really*? Ask the **why of each side**. Two things collide
   only if they're mutually exclusive *and* terminal; in Cairn almost nothing is, so
   most apparent conflicts are undrawn distinctions. Record the distinction, or the
   genuine conflict.
5. **Do we back up?** Given the above: proceed as-is, revise, or abandon and
   re-question? Name the disposition.

## Routing

- **Hardened → forward:** proceed to `/sorted` with the design stronger for it.
- **Revise → back:** return to the design / `/intent`; the change is cheaper now
  than after casting.
- **Abandon → out:** if it can't survive the pass, it doesn't get cast. That's a
  win, not a loss (CP2).

## Also: the cadence use

The same nexus runs on any *settled* artifact on a cadence — CLAUDE.md is the
most-challenged artifact in the system by charter. Same machine, slower tempo: the
question corpus accumulates yield either way (challenge-at-tempo now, background
device later). Log what the pass caught.

## Stay honest

- Don't stage a fake challenge to rubber-stamp a design you're attached to. If you
  can't find a real objection, say so plainly — that's a valid (measured) result.
- CP1: "I don't know if there's prior art" beats inventing a confident "no."
